Guía

Cómo instalar OutfitKit, tematizarlo con tokens y cómo se relaciona con Ionic.

Ionic es la base. OutfitKit cubre los huecos que Ionic no trae (tabla rica, árbol, kpi/stat, stepper/wizard, feedback inline, estados vacíos, chrome web). Para botones, inputs, listas, modales, toolbars y layout de app: usa Ionic directo — este mismo showcase está construido así.

Instalación

Instala el paquete y sus peers (Ionic y Lit los aporta el host una sola vez):

npm i @outfitkit/core
# peers
npm i @ionic/core lit

Carga de Ionic (lo registra el host)

OutfitKit asume que @ionic/core ya está registrado; no lo importa por componente. Cárgalo una vez en tu app:

import '@ionic/core';
import '@ionic/core/css/ionic.bundle.css';

Imports individuales (recomendado)

Importa solo los componentes que uses (mejor tree-shaking). El import tiene efecto secundario: registra el custom element.

import '@outfitkit/core/ok-data-table';
import '@outfitkit/core/ok-tree';
import '@outfitkit/core/ok-kpi';

// También puedes importar las clases/tipos desde el barrel:
import { OkDataTable } from '@outfitkit/core';

Bundle completo (CDN / prototipos)

Un único import que auto-registra todos los ok-*:

import '@outfitkit/core/cdn';

Theming por tokens --ok-*

OutfitKit no define valores por defecto: tú decides de dónde salen. Lo más cómodo es heredarlos de Ionic (--ion-*), con lo que el tema claro/oscuro se aplica solo. Copia @outfitkit/core/theme.css a tu CSS y ajusta lo que quieras: 

:root {
  --ok-primary: var(--ion-color-primary);
  --ok-primary-contrast: var(--ion-color-primary-contrast);
  --ok-bg: var(--ion-background-color);
  --ok-surface: var(--ion-card-background);
  --ok-text: var(--ion-text-color);
  --ok-border: rgba(var(--ion-text-color-rgb), 0.12);
  --ok-radius: 12px;
  --ok-spacing: 1rem;
  --ok-container-max: 1140px;
  --ok-font: system-ui, -apple-system, 'Segoe UI', Roboto, sans-serif;
}

Cada componente también expone vars propias estilo Ionic (sin guion bajo: --background, --color, --border-radius…) que degradan en cadena --ok-* → --ion-* → hex, así que puedes ajustar uno a uno sin tocar los tokens globales.

Sin Ionic (p.ej. una landing Django sin Ionic) pon valores propios en los tokens --ok-*.

Reglas

  • Solo Web Components (Lit, Shadow DOM). CSP estricta: sin eval / new Function (script-src 'self').
  • Prefijo de tags ok-*; eventos propios normalizados a ok-*.
  • Para arrays/objetos/funciones (columns, rows, nodes, steps…) usa property binding por JS (el.columns = […]), no atributos.
  • El resto de UI (botones, inputs, listas, layout de app) es Ionic directo.
